# Static Frontend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a GitHub Pages static site with domain search (448K entries) and 5 client-side blockchain investigation tools calling Etherscan directly from the browser.

**Architecture:** Vanilla HTML/JS/CSS on GitHub Pages. A Python build step generates a compressed search index from YAML entries. Trace tools are clean-room JS ports of CrimsonVector playbooks — user provides their own Etherscan API key (BYOK), stored in browser localStorage, calls go directly to api.etherscan.io.

**Tech Stack:** Vanilla HTML/JS/CSS, Pico CSS, GitHub Pages, Python (build step only)

**Design Spec:** `docs/superpowers/specs/2026-05-29-static-frontend-design.md`

**Security Note:** All dynamic HTML rendering uses `textContent` or the `UI.esc()` helper for user/API data to prevent XSS. The `UI.esc()` function creates a text node via `document.createElement('div').textContent = str` and returns the escaped HTML. Etherscan links use only validated hex address/hash strings. innerHTML is used only for static markup templates where all variable data is escaped first via `UI.esc()`.

## Task 3: Shared UI Components (ui.js)

**Files:**
- Create: `site/js/ui.js`

- [ ] **Step 1: Implement UI object**

The `UI` object provides:
- `getApiKey()` — reads from input field or localStorage
- `saveApiKey()` — saves input value to localStorage
- `initApiKey()` — restores saved key to input, wires save button
- `showLoading(containerId)` — shows spinner with `aria-busy="true"`
- `showError(msg, containerId)` — shows escaped error message
- `clear(containerId)` — clears container
- `esc(str)` — HTML-escapes string via textContent
- `renderTable(data, columns, containerId)` — renders table from array of objects with column definitions
- `addrLink(addr)` — short address with etherscan.io link (addr validated as hex before use in href)
- `addrLinkFull(addr)` — full address with etherscan.io link
- `txLink(hash)` — short tx hash with etherscan.io link
- `severityBadge(severity)` — colored badge span
- `tagPills(tags)` — array of tag pill spans
- `ethVal(wei)` — wei to ETH string
- `tokenVal(raw, decimals)` — raw to token string

All methods that render HTML escape user data through `esc()` before insertion. Address/hash values used in etherscan.io URLs are validated to match `/^0x[0-9a-fA-F]+$/` before inclusion.

## Task 4: Etherscan Client & Known Addresses (common.js)

**Files:**
- Create: `site/js/common.js`

- [ ] **Step 1: Implement Etherscan client, known addresses, event signatures, utilities**

**`Etherscan` object:**
- `BASE_URL`: `https://api.etherscan.io/v2/api`
- `CHAIN_ID`: 1
- Rate limiting: 220ms between requests (5/sec with key), tracks `_lastRequest` timestamp
- `call(module, action, params)` — builds URL, adds chainid + apikey, fetches, handles rate limit retries (sleep 2s, retry once), returns `data.result`
- Convenience methods: `getNormalTxs`, `getInternalTxs`, `getERC20Transfers`, `getLogs`, `getContractABI`, `getContractSource`, `getStorageAt`, `ethCall`, `getTxReceipt`

**`KNOWN_ADDRESSES` object:**
Generic entries only — CEX wallets (Binance x6, Coinbase x8, FTX x2, Kraken x3, Gate.io x3, Gemini, KuCoin), bridges (Base, Wormhole, Polygon x2, Arbitrum, Optimism, Stargate, CCTP TokenMessenger), DEX routers (Uniswap V2/V3/Universal, SushiSwap, 1inch V4/V5, 0x, Curve), mixers (Tornado Cash Router/Proxy/0.1/1/10/100 ETH), stablecoins (USDC, USDT, DAI).

NO case-specific addresses. NO personal names. NO org identifiers.

**Event signatures:** TRANSFER, ROLE_GRANTED, ROLE_REVOKED, OWNERSHIP_TRANSFERRED, UPGRADED, ADMIN_CHANGED, MESSAGE_RECEIVED, MINT_AND_WITHDRAW

**Role hashes:** MINTER_ROLE, DEFAULT_ADMIN_ROLE, PAUSER_ROLE

**CCTP constants:** TOKEN_MESSENGER, MESSAGE_TRANSMITTER addresses, domain ID map (0=Ethereum through 7=Polygon)

**Utility functions:** `normalize(addr)`, `tagAddress(addr)`, `isNullAddress(addr)`, `weiToEth(wei)`, `weiToToken(wei, decimals)`, `shortAddr(addr)`, `tsToISO(timestamp)`, `topicToAddr(topic)`, `EIP1967` storage slot constants

Clean-room port of `fund_flow.py`. Fetches normal txs, internal txs, and ERC-20 transfers in parallel via `Promise.all`. Builds node graph with entity tagging via `tagAddress()`. ## Task 8: Gas Tracer Playbook (gas-tracer.js)

**Files:**
- Create: `site/js/playbooks/gas-tracer.js`

- [ ] **Step 1: Implement GasTracer.trace(address, maxDepth)**

Clean-room port of `gas_tracer.py`. Recursively traces ETH funding chain: for each address, finds the first ETH inflow (oldest normal tx where to=address and value>0), records the funder, then recurses on the funder up to maxDepth (default 3, max 5). Stops recursion when funder is a known entity or already visited. Records early inflows (first 20) for the initial target. Falls back to internal transactions if no normal tx funding found.

Renders results as: tree view with indentation by depth, detailed funding chain table, early inflows table for the first hop.
