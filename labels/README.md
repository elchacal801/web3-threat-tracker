# Labels Dataset

First-class address labels for on-chain entity tagging.

## Schema

All CSV files in this directory follow the same schema:

| Column       | Type   | Description                                                    |
|-------------|--------|----------------------------------------------------------------|
| `chain`     | int    | Chain ID (1 = Ethereum mainnet)                                |
| `address`   | string | Lowercase hex address (with 0x prefix)                         |
| `entity`    | string | Human-readable entity name                                     |
| `category`  | string | One of: `cex`, `dex`, `bridge`, `mixer`, `sanctioned`, `token`, `defi`, `nft` |
| `source`    | string | Provenance: `manual`, `ofac`, etc.                             |
| `confidence`| string | `high`, `medium`, `low`                                        |

## Files

- **entities.csv** -- Manually curated base labels (migrated from `KNOWN_ADDRESSES` in common.js).
- **ofac.csv** -- OFAC SDN sanctioned addresses, generated by `scripts/ingest/ingest_ofac.py`.

## Build

Run `scripts/build_labels.py` to merge all CSV files into `site/data/labels.json`, which the frontend loads at runtime.
